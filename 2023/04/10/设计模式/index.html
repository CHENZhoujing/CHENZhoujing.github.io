设计模式
Apr 10 2023

六大原则

  • 单一职责原则
    • 一个类应该只有一个职责
  • 开放封闭原则
    • 对扩展开放，对修改封闭
  • 里氏替换原则
    • 字类应该可以完全替代父类
  • 接口隔离原则
    • 一个接口应该只包含客户端需要的方法
  • 依赖倒置原则
    • 依赖抽象而不是具体实现
  • 迪米特法则
    • 一个对象应该对其他对象保持最少的了解

设计模式分类

  1. 创建型模式
    1. 主要用于对象的创建和初始化
    2. 包括单例模式、工厂模式、抽象工厂模式、建造者模式和原型模式。
  2. 结构型模式
    1. 主要用于描述如何将类或对象组合成更大的结构
    2. 包括适配器模式、桥接模式、装饰器模式、组合模式、外观模式和享元模式。
  3. 行为型模式
    1. 主要用于描述对象之间的通信方式和协作方式
    2. 包括策略模式、模板方法模式、观察者模式、迭代器模式、责任链模式、命令模式、备忘录模式、状态模式、访问者模式和中介者模式。

常见设计模式

  • 单例模式：确保一个类只有一个实例，并提供全局访问点。比如全局配置对象、数据库连接池。
  • 工厂模式：定义一个用于创建对象的接口，让子类决定实例化哪个类。比如生产不同品牌手机的工厂、不同类型产品的工厂。
  • 观察者模式：定义一种一对多的依赖关系，当一个对象状态发生改变时，所有依赖它的对象都得到通知并自动更新。比如事件监听器、邮件订阅。
  • 装饰器模式：动态地给一个对象添加一些额外的职责，就像在墙上贴壁纸一样。比如加密装饰器、缓存装饰器。
  • 代理模式：为另一个对象提供一个替身或占位符，以控制对该对象的访问。比如网络代理、安全代理、远程代理。

单例模式

用于确保一个类只有一个实例，并提供全局访问点以便于使用该实例。
在饿汉式单例模式中，单例实例在类加载时就被创建，因此也被称为静态单例模式。 

public class Singleton {
    private static Singleton instance = new Singleton();
    private Singleton() {}

    public static Singleton getInstance() {
        return instance;
    }
}

在懒汉式单例模式中，单例实例在第一次使用时被创建。

public class Singleton {
    private static Singleton instance;
    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

双重检查锁定是一种常用的线程安全的单例模式实现方式。该方式首先检查是否已经存在单例对象，如果不存在，则进行加锁创建对象，再进行一次检查是否已经存在单例对象，如果还不存在，就创建一个新的对象。该方式的优点是既能保证线程安全，又能避免每次获取对象都进行加锁，提高了效率。

public class Singleton {
    private static volatile Singleton instance;
    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

遇到过哪些设计模式？

  1. 代理模式：另一个对象提供一个替身或占位符，以控制对该对象的访问 spring AOP就是基于代理模式实现。
  2. 工厂模式：定义一个用于创建对象的接口，让子类决定实例化哪个类 spring BeanFactory基于工厂模式实现。
  3. 单例模式：
    1. 数据库连接池：在多线程环境下，数据库连接池需要保证只有一个实例，否则会导致连接池资源被耗尽。
    2. 日志记录器：在多个线程中记录日志信息时，需要保证日志记录器是唯一的，以避免数据被覆盖或丢失。
