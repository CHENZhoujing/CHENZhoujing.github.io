七层网络体系结构

  1. 应用层：提供程序间通信，提供为应用软件而设的接口
    • FTP文件传输协议,
    • SMTP是建立在FTP文件传输服务上的一种邮件服务
    • POP3 即邮局协议版本3，是一种用来从邮件服务器上读取邮件的协议
    • HTTP
    • DNS 将域名和IP地址相互映射的一个分布式数据库，能够使人更方便地访问互联网
  2. 表示层：把数据转换为能与接收者的系统格式兼容并适合传输的格式，数据加密和解密
  3. 会话层：管理主机间的会话进程，包括建立、管理以及终止进程间的会话
  4. 传输层：负责主机中两个进程之间的通信，把传输表头加至数据以形成数据段
    • TCP：
    • UDP：
  5. 网络层：提供寻址和路由选择的功能，决定数据的路径选择和转寄，具有一定的拥塞控制能力，将传输层传下来的报文段封装成数据包
    • 路由器
    • ICMP：互联网控制消息协议
    • ARP：地址解析协议，是用来将IP地址解析为MAC地址的协议
    • IP：
  6. 数据链路层：将网络层传输下来的IP数据报组装成帧
    • 网桥，交换机
    • 通过各种控制下协议，把物理层封装成无差错能可靠传输数据帧的数据链路
    • 汉明码
  7. 物理层：将数据转换为可通过物理介质传送的电子信号
    • 中继器，集线器
    • 曼切斯特编码
    • 4B/5B

为什么分七层：支持异构网络的互联互通。

加密是在哪一层？获取主机IP地址是在哪一层？TCP报文头部包含那些信息？

  1. 加密通常是在表示层和应用层完成的。在表示层，可以使用 SSL（Secure Sockets Layer）或 TLS（Transport Layer Security）协议对数据进行加密，以确保数据在传输过程中的安全性。在应用层，许多应用程序（例如 Web 浏览器、电子邮件客户端）也提供加密选项，以保护用户的数据和隐私
  2. 获取主机 IP 地址通常是在网络层（第三层）中完成的。当主机连接到网络时，它会通过 DHCP（Dynamic Host Configuration Protocol）或静态配置等方式获得一个 IP 地址。
    • 源端口和目标端口：指示发送端和接收端的端口号。
    • 序列号和确认号：用于 TCP 协议的可靠传输机制，保证数据的有序性和可靠性。
    • 数据偏移量：指示 TCP 报文头的长度。
    • 控制位：包括 URG（紧急指针）、ACK（确认）、PSH（推送）、RST（重置）、SYN（同步）、FIN（结束）等控制信息。
    • 窗口大小：指示接收方缓冲区的大小，用于流量控制。
    • 校验和：用于检验 TCP 报文头和数据的正确性和完整性。
    • 紧急指针：用于指示紧急数据的位置。

TCP 和 UDP 的区别和使用场景？

  • TCP是一种可靠的、面向连接的、有序的协议，适用于需要确保数据完整性和可靠性的场景，如文件传输、邮件发送等。
  • 而UDP是一种不可靠的、非连接性的、无序的协议，适用于实时传输、流媒体、游戏等场景，因为它具有高效、快速的特点。
  • TCP和UDP对消息大小的限制不同。TCP将数据分割成固定大小的数据块（通常为MTU大小），然后再发送，而UDP则没有这样的限制，可以发送任意大小的数据。

TCP三次握手四次挥手

  • 三次握手
    • 客户端向服务端发送 SYN 报文，表示要建立连接，SYN 标志位被置为 1，随机生成一个初始序列号 seq。
    • 服务端接收到 SYN 报文后，向客户端发送 SYN+ACK 报文，表示确认收到连接请求，并发送自己的 SYN 报文，SYN 和 ACK 标志位都被置为 1，确认号 ack 等于客户端的 seq+1，同时服务端也随机生成一个初始序列号 seq。
    • 客户端接收到 SYN+ACK 报文后，向服务端发送 ACK 报文，确认收到服务端的确认，ACK 标志位被置为 1，ack 等于服务端的 seq+1，表示下一个序列号从服务端的初始序列号开始。

  • 四次挥手
    • 客户端向服务端发送 FIN 报文，表示要关闭连接，FIN 标志位被置为 1，seq 等于当前的数据流的最后一个字节的序号加一。
    • 服务端接收到 FIN 报文后，向客户端发送 ACK 报文，确认收到关闭请求，ack 等于客户端的 seq+1。
    • 服务端向客户端发送 FIN 报文，表示服务端也要关闭连接，FIN 标志位被置为 1，seq 等于当前的数据流的最后一个字节的序号加一。
    • 客户端接收到服务端的 FIN 报文后，向服务端发送 ACK 报文，确认收到关闭请求，ack 等于服务端的 seq+1。此时，客户端进入 TIME_WAIT 状态，等待 2 倍的 MSL（最大报文生存时间）时间后才会关闭连接。

HTTP 和 HTTPS 的区别

HTTP是明文传输，端口号80，HTTPS则会有加密，需要到ca申请证书，端口号443。

具体讲讲HTTPS的过程

  • 客户端发送发送一个加密连接请求给服务器，其中包含客户端支持的TSL/SSL版本和支持的加密算法。
  • 服务器回应一个数字证书，证书中包含了最终选择的算法和TSL版本，以及非对称加密的公钥。
  • 客户端收到证书之后会对证书进行验证，如果可信，客户端会生成一个协商密钥，用证书中的公钥就行加密，并发送给服务端。
  • 服务端收到加密后的协商密钥之后，使用私钥进行解密，这样子就得到了协商密钥。并发送一个消息给客户端，通知其已经可以加密对话了。

HTTP 1.0 1.1 2.0

  • HTTP 1.0：每个请求都需要建立一个新的连接，请求完成后立即关闭连接。这
  • HTTP 1.1：引入了持久连接（Persistent Connection）的概念，即在一个连接上可以发送多个请求和响应。
  • HTTP 2.0：
    • 引入了多路复用（Multiplexing）的概念，即在一个连接上可以同时发送多个请求和响应，不再需要按顺序等待响应。这种方式可以进一步提高性能。
    • 引入了头部压缩（Header Compression）的机制，可以减少重复的头部信息，降低传输数据的大小。

HTTP 状态码分类

信息响应(100–199)，成功响应(200–299)，重定向(300–399)，客户端错误(400–499)和服务器错误 (500–599)：

  • 200 - 请求成功
  • 301 - 资源（网页等）被永久转移到其它URL
  • 404 - 请求的资源（网页等）不存在
  • 500 - 内部服务器错误

Cookies Sessions区别

  • Cookie保存在客户端，大小为4k，安全性低，session保存在服务端，大小可以很大，安全性高。
  • 有效期不同，Cookie可设置为长时间保持，比如默认登录功能功能，Session一般有效时间较短，客户端关闭或者Session超时都会失效。有效期不同，Cookie可设置为长时间保持，比如默认登录功能功能，Session一般有效时间较短，客户端关闭或者Session超时都会失效。
  • 大多数的应用都是用Cookie实现Session跟踪的。第一次创建Session时，服务端会通过在HTTP协议中返回给客户端，在Cookie中记录SessionID，后续请求时传递SessionID给服务，以便后续每次请求时都可分辨你是谁。
  • 禁用cookie怎么办？拼接SessionId参数。在GET或POST请求中拼接SessionID，GET请求通常通过URL后面拼接参数来实现，POST请求可以放在Body中.
