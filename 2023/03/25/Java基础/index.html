BIO/NIO/AIO

  • Blocking IO同步阻塞，一个连接一个线程，即客户端有请求是服务端就会启动一个线程来处理，如果这个连接不做任何事情，则会造成额外开销，可以通过线程池来改善。
  • Non-Blocking IO同步非阻塞，客户端发送的连接请求都会注册到多路复用器上，多路复用器轮询到连接有 I/O 请求时才启动一个线程进行处理。
    • JDK 的 NIO 底层由 epoll 实现。
  • Asynchronous-Non-Blocking-IO异步非阻塞，线程发起 IO 请求，立即返回；内存做好 IO 操作的准备之后，做 IO 操作，直到操作完成或者失败，通过调用注册的回调函数通知线程做 IO 操作完成或者失败。

使用场景

  • BIO：连接数比较小且固定，JDK1.4 之前唯一的选择。
  • NIO：连接数多且连接比较短，比如聊天服务器，弹幕系统，服务器间通讯等，JDK1.4开始引入。
  • AIO：连接数多且连接比较长，比如相册服务器，JDK7开始引入。

例子

  • 同步阻塞：你到饭馆点餐，然后在那等着，啥都干不了，餐没做好，你就必须等着！
  • 同步非阻塞：你在饭馆点完餐，就去玩儿了。不过玩一会儿，就回饭馆问一声：好了没？
  • 异步非阻塞：饭馆打电话说，我们知道您的位置，一会给你送过来，安心玩儿就可以了，类似于外卖。

面向对象

  • 对象：对象是人们要研究的任何事物，他不仅能表示具体的事物，也能表示抽象的规则，计划或者事件。对象是数据和操作的结合，使数据和操作封装在统一的对象中。
  • 类；具有相同特性和操作的的对象的抽象就是类。对象的抽象是类，类的具体化就是对象。
  • 三大特性：
    • 封装：让某个类型的对象获得另一个类型的对象的属性的方法。
    • 继承：隐藏部分对象的属性和实现细节，对数据的访问只能通过外公开的接口。
    • 多态：对于同一个行为，不同的子类对象具有不同的表现形式。
    • 多态存在的3个条件：1）继承；2）重写；3）父类引用指向子类对象。

JDK1.8之后有哪些新特性？

  1. Lambda 表达式和函数式接口：Lambda 表达式本质上是一段匿名内部类，也可以是一段可以传递的代码。Lambda 允许把函数作为一个方法的参数（函数作为参数传递到方法中），使用 Lambda 表达式使代码更加简洁，但是也不要滥用，否则会有可读性等问题，
  2. Stream API：用函数式编程方式在集合类上进行复杂操作的工具，配合Lambda表达式可以方便的对集合进行处理。Java8 中处理集合的关键抽象概念，它可以指定你希望对集合进行的操作，可以执行非常复杂的查找、过滤和映射数据等操作。使用Stream API 对集合数据进行操作，就类似于使用 SQL 执行的数据库查询。也可以使用 Stream API 来并行执行操作。简而言之，Stream API 提供了一种高效且易于使用的处理数据的方式。
