单例模式

用于确保一个类只有一个实例，并提供全局访问点以便于使用该实例。
在饿汉式单例模式中，单例实例在类加载时就被创建，因此也被称为静态单例模式。 

public class Singleton {
    private static Singleton instance = new Singleton();
    private Singleton() {}

    public static Singleton getInstance() {
        return instance;
    }
}

在懒汉式单例模式中，单例实例在第一次使用时被创建。

public class Singleton {
    private static Singleton instance;
    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

双重检查锁定是一种常用的线程安全的单例模式实现方式。该方式首先检查是否已经存在单例对象，如果不存在，则进行加锁创建对象，再进行一次检查是否已经存在单例对象，如果还不存在，就创建一个新的对象。该方式的优点是既能保证线程安全，又能避免每次获取对象都进行加锁，提高了效率。

public class Singleton {
    private static volatile Singleton instance;
    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

工厂模式

工厂模式将对象的创建交给专门的工厂类，客户端不再直接创建对象，而是通过调用工厂类的方法来创建对象。
